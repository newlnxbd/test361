经典面试：

  问：描述嵌入式Linux的启动流程？
  
  上电 → ROM代码(SoC固化) → Bootloader(SPL/U-Boot) → 内核 → Init → 用户空间
  
  1. ROM代码：固化在SoC中，初始化最小硬件（如SRAM），从Flash/SD/eMMC加载SPL
  2. SPL（二级程序加载器）：初始化DRAM，将完整U-Boot加载到DRAM
  3. U-Boot：初始化外设，将内核 + DTB + (可选)initramfs加载到内存，通过bootargs传递启动参数，跳转到内核入口
  4. 内核：自解压，建立MMU/页表，初始化各子系统（调度器、内存、驱动），挂载根文件系统，执行/sbin/init（PID 1）
  5. Init系统：systemd或BusyBox init——启动服务、挂载文件系统、启动网络
  
  问：什么是设备树（Device Tree）？
  
  一种数据结构（.dts源文件 → .dtb二进制文件），在运行时向内核描述硬件信息，取代了硬编码的板级文件。
  
  - 描述内容：CPU、内存、总线、外设、中断、时钟、GPIO、引脚复用
  - 内核启动时解析DTB来发现和配置硬件
  - 允许一个内核二进制支持多块开发板

  问：U-Boot如何向内核传递信息？
  
  - ARM：内核入口时 r0=0，r1=机器ID，r2=DTB地址
  - ARM64：x0=DTB地址
  - 内核命令行：存储在DTB的/chosen节点中或U-Boot的bootargs环境变量中
  - 示例：bootargs=console=ttyS0,115200 root=/dev/mmcblk0p2 rootwait rw

  问：主设备号和次设备号？
  
  - 主设备号：标识驱动程序（如4 = tty驱动）
  - 次设备号：标识该驱动管理的具体设备实例（如ttyS0、ttyS1）
  - 现代做法：alloc_chrdev_region() 动态分配，udev自动创建设备节点

  问：kmalloc、vmalloc、kzalloc的区别？
  
  ┌─────────┬──────────────────┬────────────────┬───────────────────────────┐
  │ 函数    │ 物理连续？       │ 大小限制       │ 使用场景                  │
  ├─────────┼──────────────────┼────────────────┼───────────────────────────┤
  │ kmalloc │ 是               │ ~4MB           │ DMA缓冲区、小分配，速度快 │
  ├─────────┼──────────────────┼────────────────┼───────────────────────────┤
  │ kzalloc │ 是（零初始化）   │ 同kmalloc      │ 同上，但内存清零          │
  ├─────────┼──────────────────┼────────────────┼───────────────────────────┤
  │ vmalloc │ 否（仅虚拟连续） │ 受地址空间限制 │ 大缓冲区，不要求物理连续  │
  └─────────┴──────────────────┴────────────────┴───────────────────────────┘
  
  - kmalloc使用slab分配器，从低端内存直接映射区分配——更快，无需建立新页表
  - vmalloc需要创建新的页表映射——更慢，不能用于DMA
  - GFP标志：GFP_KERNEL（可睡眠），GFP_ATOMIC（不可睡眠，中断上下文中使用）

  问：什么是initramfs/initrd？
  
  - 一个临时根文件系统，由Bootloader和内核一起加载到内存
  - 内核将其挂载为初始根文件系统，运行/init脚本
  - 用途：加载访问真正根设备所需的驱动（如USB、RAID、加密磁盘、NFS）
  - 完成后，switch_root切换到真正的根文件系统，释放initramfs占用的内存
  
  # 创建initramfs：
  find . | cpio -o -H newc | gzip > initramfs.cpio.gz
  
  # U-Boot中加载：
  load mmc 0:1 ${loadaddr} zImage
  load mmc 0:1 ${rdaddr} initramfs.cpio.gz
  load mmc 0:1 ${fdtaddr} board.dtb
  bootz ${loadaddr} ${rdaddr}:${filesize} ${fdtaddr}
  
  - initramfs（现代）：cpio归档，解压到tmpfs
  - initrd（传统）：文件系统镜像（如ext2），作为块设备加载到/dev/ram0
